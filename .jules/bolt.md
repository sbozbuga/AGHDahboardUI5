## 2024-05-18 - Native Number() is Faster than typeof + parseFloat
**Learning:** In V8 engines, the combination of `typeof ms === 'number' ? ms : parseFloat(ms)` is significantly slower (~6x slower in tight loops) than the native `Number()` function. This is because `Number()` has highly optimized built-ins for type coercion, whereas `parseFloat` forces a string parsing pathway even when the input could be more simply coerced, and the ternary check adds branching overhead.
**Action:** Used `Number()` instead of manual type checking and `parseFloat` for numeric conversions in high-frequency functions like `formatter.ts` and loop iterations in `StatsService` and `LogService`. Note: `Number()` coercion behaves differently for null/undefined/empty string compared to `parseFloat`, so ensure `ms == null` is checked explicitly beforehand.
